Types of Servers

To set up a server, all you're really going to need a few key components: A server, plugins and some friends.

In order to get started, you need to decide which type of server you require. For OCN, there are two types you might want to download. You can choose to either create a map developing server or a scrimmage server. Map developing servers are used to create maps that you will hope to be loaded onto public rotations, for the Overcast Netowork. Comparatively, scrimmage servers are for practicing team strategies and perhaps your own strategies with friends, or for hosting scrimmages.

To set up a server, you're going to need a few key components: A server, plugins and some friends.

Map Developing Servers

Downloading (Server Jar and Plugins)

To start, you're going to need to decide upon a server jar. As Minecraft's downloadable server services do not allow plugins, you will have to download a different type. Bukkit is currently disfunyct so we cannot use that either. That leaves us with Spigot and alternate bukkit clones. Spigot is the general replacement for Bukkit and what is reccommended for map developing. It's downloading guide is here. However if you prefer, you can download an alternate Bukkit clone. For the ease of further steps, your downloaded jar file should be renamed to server.jar.

Also, plugins need to be downloaded. For map making there is some reccommended plugins that are commonly used, along with some good ones that are generally not used. These plugins will be listed below along with a short description and download link.

  • Essentials. An extension on basic commands. This plugin supplies a huge arsenal of commands ranging from /gm to /kittycannon which admitedly, aren't all useful, but the good heavily outweighs the bad. This is a must get for any server. Just a heads up, the plugin jars are included in the jar you download.Download Essentials here.
  • World Edit. Everyone's heard of this plugin, and for a good reason. It easily allows you to flip, rotate and manipulate maps with simple commands. Highly reccommended for mapmaking servers. World Edit does have some problems currently, so if you download I reccommend looking for a fix for it as well, there's a few out there but they're hard to find, or just don't use those commands. Use /flip instead of /rotate. Download World Edit here.
  • World Edit Visualiser. A great add-on for World Edit. It shows your selections with an outline of redstone particles so you know what you're selecting. Very under-appreciated. Download World Edit here.
  • Multiverse. A must have. Easily create worlds with special properties, and easily manipulate world properties Download Multiverse here.
  • Voxel Sniper. This is another pretty common one, but not many people know how to use it. It's basically a long range editting tool, with command that do similar things to World Edit. If you're going to use it, research it a bit and read the Curse description before deciding to download. Download Voxel Sniper here.
  • World Guard. Now this plugin doesn't really have the best rap, but it's very useful for map making. You can change effects, block updating and much more. This includes vine/crop growth, fire spread and stopping water from flowing. Most used command is usually /stoplag which stops all block updates server-wide and removes all entities. Download World Guard here.
  • Maps. You're also going to want to download maps. Maps have to be made with correct XML and files. There is a guide for this here. Alternatively, you can download maps from the Overcast Network repository which can be found here.

Starting your server.

To start your server you're going to need to create a text file, to allocate RAM and to create a start script. This text file will change depending on what you're operating system is. Here is a windows one and an apple one, sorry Linux users, time to go research it. It's important that you've renamed your server jar to server.jar.

Windows Users:
@echo off
java -Xmx(NUMBER OF MEGABYTES YOU WANT TO ALLOCATE)M -XX:MaxPermSize=128M -jar server.jar
pause
Then name this text file run.bat and save it in the same place as your server jar. It's probably smart to put them both in a new folder.

Mac Users:
#!/bin/bash
cd "$(dirname "$0")"
exec java -Xmx(NUMBER OF MEGABYTES YOU WANT TO ALLOCATE)M -jar server.jar nogui
Then name the text file run.command and save it in the same place as your server jar. It's probably smart to put them both in a new folder.

Now, run your text file (which will now be either a batch file or a command script).

Setting the server up.

If you haven't already, and if it's done loading, feel free to close your server that should have opened. If you look back in your folder with the jar file in it, you'll now have a whole server directory. Your plugins folder should be there, so take a second and put all of your downloaded plugin jars from before into the folder. Now run it again and your server will be fully functional!

Hosting

The last thing you need to consider is how you're going to host your server.
You can either host it on your own computer, and only put it up when you need it, or you can pay for hosting. Beastnode have good deals ranging through everything you could need.You can find there deals here.

If you want to host on your own computer, you need to do a few things including port forwarding, and getting a static IP adress. The port forwarding is mandatory however the static IP is not, but your IP will change frequently. To port forward, a guide can be found here. That website also has a static IP walkthrough. It is very helpful if you're going to try to port forward yourself.
